ChemDCAT-AP Widget

An interactive, schema-driven metadata editor for chemical dataset descriptions — built on ChemDCAT-AP / LinkML.

▶ Launch Widget Read the guide ChemDCAT-AP Docs ↗
🔗

Schema-driven

Nodes and fields are generated from the dcat-4C-ap LinkML schema. Load any schema class as an editable node on the canvas.

📐

Quantitative attributes

Measurement fields carry value, unit, and auto-derived has_quantity_type (QUDT URIs) — fully round-trippable.

↔️

Import / Export

Load existing LinkML instance JSON, build or extend the metadata graph, then export as JSON or YAML — ready for LinkML generators.

🧩

Embeddable

Designed as a standalone web component — embed via <iframe> in ELNs, documentation sites, or data portals.

Quick start

Getting started with the widget takes just a few clicks.

  1. 1

    Open the widget

    Click "▶ Launch Widget" above or navigate to the Try it page. The welcome screen shows example datasets to explore.

  2. 2

    Load an example or start fresh

    Click one of the example cards (e.g. MaterialSample) to load a pre-built metadata graph. Or click "New canvas" and drag schema classes from the palette on the left.

  3. 3

    Edit nodes and connect them

    Click any node to expand its fields. Fill in text, pick units for measurements, and drag handles between nodes to create relationships. Use has_mass, has_temperature, and other measurement slots directly.

  4. 4

    Export to LinkML

    Click ↓ JSON or ↓ YAML in the toolbar to download a LinkML-compatible instance document. Feed it to linkml-convert, gen-jsonld, or any other LinkML generator.

  5. 5

    Round-trip import

    Use 📂 Load in the toolbar to re-import a previously saved JSON file. The canvas reconstructs all nodes and edges from the saved instance document.
